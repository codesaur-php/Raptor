Хэрэглэгч гэж юу вэ?

Users нь системийн хэрэглэгчдийг удирдах модуль юм. Хэрэглэгчийн бүртгэл үүсгэх, засварлах, устгах зэрэг бүх үйлдлийг энэ модулиар хийнэ.

  • Бүртгэл - Шинэ хэрэглэгч үүсгэх, мэдээлэл бүртгэх
  • Засварлах - Хэрэглэгчийн мэдээллийг өөрчлөх
  • Устгах - Хэрэглэгчийг идэвхгүй болгох (soft delete)
  • Байгууллага - Хэрэглэгчийг байгууллагад оноох
  • RBAC дүр - Хэрэглэгчид эрхийн дүр (role) оноох
  • Нууц үг - Нууц үг удирдлага

Бүртгүүлэх хүсэлт товчийг дарахад modal цонх нээгдэж, вэб сайтаас бүртгүүлэх хүсэлт илгээсэн хэрэглэгчдийн жагсаалт харагдана.

Үйлдлүүд

Зөвшөөрөхөд system_user_insert, устгахад system_user_delete эрх шаардлагатай.

Нууц үг шинээр тааруулах

Нууц үг шинээр тааруулах товчийг дарахад modal цонх нээгдэж, нууц үг мартсан хэрэглэгчдийн хүсэлтүүдийн жагсаалт харагдана.

ХэрэглэгчНэр, username, ID
ХэлХэрэглэгчийн хэл (тугаар)
Төлөвready - ашиглах боломжтой, used - хэрэглэгдсэн, expired - хугацаа дууссан

Засварлах

Жагсаалтаас засварлах товчийг дарна.

  • Бүх талбаруудыг өөрчлөх боломжтой (username-аас бусад)
  • Зураг солих эсвэл устгах боломжтой

Байгууллага оноох

system_user_organization_set эрхтэй бол хэрэглэгчийг байгууллагад нэмэх эсвэл хасах боломжтой.

system_coder дүртэй хэрэглэгч бүх байгууллагад нэвтрэх эрхтэй.

RBAC дүр оноох

system_rbac эрхтэй бол хэрэглэгчид дүр (role) оноох эсвэл хасах боломжтой. (Дэлгэрэнгүй: RBAC гарын авлага)

system_coder дүртэй хэрэглэгч бүх үйлдэл хийх эрхтэй.

Нууц үг

  • Өөрийн нууц үгийг солих боломжтой
  • system_coder эрхтэй бол бусдын нууц үг солих боломжтой

Хэрэглэгчийн бүртгэлтэй холбоотой дараах үйлдлүүд Discord webhook-ээр мэдэгдэл илгээнэ (webhook тохируулсан бол):

Хэрэглэгчийн бүртгэлтэй холбоотой автомат и-мэйлүүд:

Нууц үг сэргээх хүсэлт Хүсэлт гаргагч forgotten-password-reset Нууц үг сэргээх линк, и-мэйл хаяг

Устгах

Жагсаалтаас устгах товчийг дарна.

  • SweetAlert баталгаажуулалт гарна
  • Шалтгаан бичих текст input талбар
  • Зөөлөн устгалт (soft delete) - өгөгдөл хадгалагдана, зөвхөн идэвхгүй болно
