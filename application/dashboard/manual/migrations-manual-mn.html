Migration гэж юу вэ?

Migration бол анхны setup-аас хойш өгөгдлийн санг өөрчлөх SQL скрипт юм: ихэвчлэн schema өөрчлөлт (ALTER TABLE, CREATE INDEX) эсвэл seed data (translations, menu гэх мэт). Шинэ хүснэгт migration-аар үүсгэхгүй - Model class бүр өөрийн хүснэгтийг анхны хэрэглээ дээр автоматаар үүсгэдэг.

Эрх Migration-д зөвхөн system_coder role-той хэрэглэгч хандах боломжтой.

Хэрхэн ажилладаг

Raptor нь file-based migration систем ашигладаг - state нь файлын disk дээрх байршлаас тогтоогдоно.

database/migrations/
  {userId}-{username}/
    pending_file.sql     <- pending
    ran/
      applied_file.sql   <- амжилттай ажилласан

database/migrations/ хавтас нь git-ээр ignore хийгдсэн. Орчин бүр өөрийн SQL файлуудаа энэ хуудаснаас upload хийнэ. Анх удаа upload хийхэд per-user folder автоматаар үүсэх ба сервер дээр хэн ямар файл upload хийсэн, аль нь амжилттай ажилласан гэх мэт түүх хадгалагдана.

SQL файлын формат

-- Эхний мөрийн тайлбар (summary болж UI-д харагдана)
ALTER TABLE products ADD COLUMN category VARCHAR(100);
CREATE INDEX idx_products_category ON products (category);
  • Statement-ууд дараалан ажиллана. Нэг нь алдаа гаргавал бусад нь ажиллахгүй, файл pending хэвээр үлдэнэ.
  • Файлын нэрлэх зөвлөмж: YYYY-MM-DD_short-description.sql (хугацаагаар sort хийгдэхүйц).
  • Хэмжээний хязгаар: min(10 MB, php.ini post_max_size, upload_max_filesize) - upload товчны tooltip-д тухайн серверийн бодит max харагдана. Зөвхөн .sql extension зөвшөөрнө.

Ажлын урсгал

  1. Дээд талын Upload SQL товчийг даран .sql файл сонгоно.
  2. Файл таны folder-д pending файл болж буудаг.
  3. View товчийг даран SQL, summary, SHA-256 hash, security warning-уудыг үзнэ.
  4. Apply товчийг дарвал warning байвал CONFIRM гэж (томоор) бичиж баталгаажуулна.
  5. Амжилттай бол файл ran/ руу шилжинэ. Амжилтгүй бол файл pending хэвээр, алдааг Logs модулиар харна.
  6. Apply хийхээсээ өмнө буруу файлыг устгахыг хүсвэл pending row дээрх Delete товчийг ашиглана.

Аюулгүй байдлын хамгаалалт

Apply хийхээсээ өмнө SQL нь дараах sensitive хүснэгт болон pattern-уудтай таарч байгаа эсэхийг шалгагдана. Таарсан тохиолдолд soft warning гарч, Apply товч CONFIRM бичүүлж шаардана:

  • users - нууц үг / identity
  • rbac_roles, rbac_permissions, rbac_user_role, rbac_role_permission - privilege escalation эрсдэл
  • organizations, organizations_users - multi-tenant хязгаар
  • localization_language - сайт даяар хэлний нөлөөлөл
  • raptor_menu - dashboard навигаци
  • GRANT / REVOKE
  • CREATE USER / DROP USER / ALTER USER
  • CREATE TABLE - Model class дээр setTable()-ээр тодорхойлох нь зөв. Raptor нь Model-ийг анх ашиглахад хүснэгтийг автоматаар үүсгэдэг.

Хамгаалалт зөөлөн (hard block биш). Зорилго нь зориудаас гадуурх privilege change-ээс сэргийлэх. Энгийн password reset эсвэл role grant хийхдээ SQL migration-аас илүү dashboard UI-г ашиглах нь зөв. Шинэ хүснэгтийн хувьд migration биш, Model-ийг сонгох ёстой.

Audit мөрдөлгөө

Upload, apply, delete бүр dashboard_log-д тэмдэглэгдэнэ: SQL-ийн SHA-256 hash, statement тоо, warning тоо, очих folder. Disk дээрх file бүтэц өөрөө audit trail юм - pending файл user folder-т, амжилттай ажилласан файл ran/-д.
