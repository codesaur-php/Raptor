What is Messages?

Messages is the Dashboard module for managing messages received from the website's contact page. You can view, mark as replied, and delete visitor messages.

Each message contains:

  • Name - Sender's name
  • Phone - Contact phone number
  • Email - Email address (optional)
  • Message - Message content
  • Status - New / Read / Replied

Messages List

Click Messages in the Dashboard sidebar to open the list page.

Filter

Filter messages by Status:

  • New - Unread message
  • Read - Read but not replied
  • Replied - Marked as replied
Table Columns

ActionsView, Reply, Delete buttons

View Message

Click the view (eye) button to open the message details in a modal.

  • Name, date, phone, and email information
  • Full message content
  • Current status (new, read, replied)
  • New messages are automatically marked as Read when opened

Mark as Replied

Click the reply button to open the reply method dialog.

Reply Method
  • Contacted by phone - If replied by phone call
  • Contacted by email - If replied by email (disabled when message has no email)

At least one method must be selected. An optional note can be added.

Messages already marked as replied will not show the reply button.

Delete Message

Click the delete (trash) button to open a confirmation dialog.

  • Permanently deleted from the database; a backup copy is stored in the Trash table for recovery
  • Requires system_content_delete permission

Admin Email Notification

An automatic email notification can be sent to an admin whenever a new message is received from the website's contact form. This setting is located at the top of the list page (visible only to users with system_coder role).

Settings
SettingDescription
Enable / Disable Toggle switch ON = sets the email address, OFF = clears the email address

Notification emails use the contact-message-notify template.

This setting directly modifies the RAPTOR_CONTACT_EMAIL_TO value in the .env file. Email set = notifications on, empty = notifications off.
