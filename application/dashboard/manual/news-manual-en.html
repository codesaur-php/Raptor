What is News?

News is the news and article management module of the Raptor Dashboard. It handles creating, editing, deleting, and publishing news articles.

Key features:

  • Multi-language - Separate articles per language
  • SEO-friendly slug - Auto-generated URL slug from title
  • Types & Categories - article, announcement, embed, gallery, video, audio, live
  • Publishing workflow - Draft to published transition

News List

Click News in the Dashboard sidebar to open the list page.

Filters

Filter news by:

  • Language - mn, en, etc.
  • Type - article, announcement, embed, etc.
  • Category - general, activity, press, event, etc.
  • Status - Published or draft

Publishing Options

Users with system_content_publish permission see these options:

  • Publish - Make visible on the website
  • Feature - Show in the homepage featured section
  • Comments - Allow visitor comments (default: enabled)
Auto-generated Fields
  • Slug - Auto-generated from title (Cyrillic to Latin transliteration)
  • Description - Auto-extracted from content (200 characters) if left empty

Edit Article

Click the edit (pencil) button from the list.

  • All fields can be modified (except slug)
  • Header image can be replaced or removed
  • Attachment descriptions can be edited, new files added, or existing files deleted
  • Editing a published article requires system_content_publish permission
  • If no changes are detected, saving is rejected ("No update!" error)
Activity Log

Users with system_logger permission can view the last 10 activity entries on the edit page. Click a log entry to see full details in a modal.

View Article

Click the view (eye) button from the list.

  • All fields displayed in read-only mode
  • Content shown in moedit read-only mode
  • Header image and attachments visible
  • Publication status and date

Delete Article

Click the delete (trash) button from the list.

  • A confirmation dialog appears (SweetAlert2) with the article thumbnail
  • Soft delete - data is preserved, only deactivated
  • Requires system_content_delete permission

Discord Notification

Reset Sample Data

When the system is first installed, sample data is pre-loaded. If sample data is detected, a yellow alert with a Reset & Production button appears on the list page.

Warning: Resetting will delete only sample data and their file attachment records. User-created content will remain intact. This action cannot be undone.
How sample data is detected: News articles with category='_raptor_sample_' are considered sample data. If you want to keep some of them as real content, simply change the category (e.g. to general) and save - they will not be deleted during reset.
