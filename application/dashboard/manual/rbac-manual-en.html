What is RBAC (Role-Based Access Control)?

A module for managing user access rights. Each user's access is determined by a combination of Roles and Permissions. Each organization has its own RBAC alias to create separate permission groups.

  • Alias system - Each organization has its own alias (e.g., system, common, university)
  • Role - A role assigned to users (admin, manager, editor, viewer, etc.)
  • Permission - Authorization to perform a specific action (user_insert, content_delete, etc.)
  • Matrix view - Managed via checkboxes in a Role x Permission table

Matrix table

The main RBAC page is displayed as a Role (row) x Permission (column) table.

  • Checking a checkbox grants the permission to that role (AJAX POST)
  • Unchecking a checkbox revokes the permission (AJAX DELETE)
  • Permissions are grouped by module (user, content, product, localization, etc.)
  • Add Role and Add Permission buttons are located at the top right

Alias

Each organization has its own RBAC alias. Roles and permissions are grouped under each alias.

  • system - The default system alias
  • common - A shared alias for multiple organizations; users in organizations with the common alias share the same access rights
  • Organization alias (e.g., university) - A separate permission group
  • Clicking an alias in the organization list navigates to the RBAC matrix (Details: Organizations manual)

Add new role

Displayed as a modal. Fields:

  • Name (name) - required, unique, up to 128 characters
  • Description (description) - optional
A role named coder cannot be created under the system alias (reserved by the system).

Add new permission

Displayed as a modal. Fields:

  • Module (module) - optional, select from existing modules or enter a new one (default: general)
  • Name (name) - required, unique, up to 128 characters
  • Description (description) - optional
A permission named system_coder cannot be created (reserved).

Permission check mechanism

When a user logs in, all roles and their permissions are loaded into memory by the RBAC system. The permission check key format is: {alias}_{name} (e.g., system_rbac, system_content_delete).

  • A user with the system_coder role has all permissions (no checks performed)

Log

If the user has the system_logger permission, a log of recent RBAC-related actions is displayed at the bottom of the page.
