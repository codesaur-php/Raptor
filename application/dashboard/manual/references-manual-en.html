What is References?

References is the reference content management module of the Raptor Dashboard. It allows managing various system texts, templates, and configurations from a single location.

Key features:

  • Multi-language - Separate title and content per language
  • Dynamic tables - New reference tables can be added automatically
  • Keyword - Each record is identified by a unique keyword
  • Categories - Grouped by categories like system, email, general, etc.

Reference Tables

The References module can work with multiple tables. Each table consists of a pair: reference_{name} and reference_{name}_content.

Each table appears as a tab on the list page.

Reference List

Click References in the Dashboard sidebar to open the list page.

  • Each table is displayed as a tab
  • URL hash (#templates-tab-pane) can be used to directly select a tab
  • Each table shows keyword, title per language, category, and action columns

View Record

Click the view (eye) button to open the record details in a modal window.

  • Keyword and category
  • Title and content for each language
  • Only existing records are displayed (deleted records are moved to Trash)

Edit Record

Click the edit (pencil) button to open the edit form in a modal window.

  • Keyword, category, title and content per language can be modified
  • If no changes are detected, saving is rejected ("No update!" error)
  • Requires system_content_update permission

Delete Record

Click the delete (trash) button from the list.

  • A confirmation dialog appears (SweetAlert2)
  • Permanently deleted from the database; a backup copy is stored in the Trash table for recovery
  • Requires system_content_delete permission

Multi-language Support

Each reference record can store content in multiple languages. When creating or editing, separate tabs appear for each language.

  • Title and Content fields per language
  • Content is stored in HTML format
  • Email templates can use {{ variable }} placeholders
LocalizedModel: Base data (keyword, category) is stored in reference_{name}, while per-language content (title, content) is stored in reference_{name}_content.
