moedit гэж юу вэ?

moedit нь Raptor Dashboard-д ашиглагддаг WYSIWYG текст засварлагч юм. Хуудас, мэдээ, бүтээгдэхүүн зэрэг контент бэлтгэхэд зориулагдсан.

Dashboard-ийн дараах хэсгүүдэд ашиглагдана:

  • Хуудсууд (Pages) - Хуудас үүсгэх, засварлах, харах
  • Мэдээнүүд (News) - Мэдээ үүсгэх, засварлах, харах
  • Бүтээгдэхүүнүүд (Products) - Бүтээгдэхүүн үүсгэх, засварлах, харах

Контентийн төрөл embed байвал автоматаар HTML эх код (Source) горимд шилжинэ. Бусад тохиолдолд Visual горимд ажиллана.

Toolbar-ын эхэнд байрлах Optimize Images checkbox идэвхтэй байвал зураг байршуулах үед автоматаар шахна (50-80%-ийн хэмжээ буурна, чанар хадгалагдана).

Хамаарах хэсгүүд:

  • Толгой зураг
  • Контент дотор оруулсан зурагнууд
  • Хавсралт файлуудаас зургийн төрлийнх

Толгой зураг (Header Image)

Мэдээ, бүтээгдэхүүн зэрэг контентийн нүүр зураг оруулахад ашиглана. Нэг контентод зөвхөн 1 толгой зураг байна.

  • Toolbar дээрх товч дарна
  • Зураг оруулсны дараа editor-ийн дээд хэсэгт харагдана
  • Зургийг солих - (Change) товч дарна
  • Зургийг устгах - (Remove) товч дарна

Зураг оруулах

Toolbar дээрх товч дарахад хоёр сонголттой цонх нээгдэнэ. Контент дотор хэдэн ч зураг оруулах боломжтой.

Зөвлөгөө:
  • Зургийг чирж (drag & drop) шууд editor дотор тавьж болно

Хавсралт файл

Toolbar дээрх товч дарж контентод файл хавсаргана. Хэдэн ч файл хавсаргах боломжтой.

Хавсралт файлуудын хүснэгтэд:

  • Файлын нэр - татах холбоостой
  • Шинж чанар - төрөл (extension), хэмжээ
  • Тайлбар - засварлах боломжтой
  • Огноо - байршуулсан огноо
  • Устгах - нэг бүрчлэн устгах боломжтой

Засварлах (Update) горимд өмнө хавсаргасан файлууд автоматаар ачаалагдана.

HTML бүтэц шалгалт

moedit нь контентын HTML бүтцийг хоёр түвшинд шалгаж, эвдэрсэн HTML-ээс хамгаална:

1. Editor дотоод шалгалт (Source -> Visual)

Source горимоос Visual горим руу шилжих үед HTML бүтцийг автоматаар шалгана:

  • Хаагдаагүй HTML comment (<!-- ... ) илрүүлнэ
  • Эвдэрсэн tag-аас болж контентын 20%-аас дээш хэсэг алга болсон бол анхааруулна
  • Асуудал илэрвэл Visual горим руу шилжилгүйгээр Source горимд үлдэж, хэрэглэгчид засах боломж олгоно
  • Built-in popup мэдэгдлээр анхааруулга харуулна
2. Сервер талын шалгалт (Хадгалах үед)

Хуудас, мэдээ, бүтээгдэхүүн хадгалах үед сервер дээр HTML контентыг дахин шалгана:

  • Хаагдаагүй HTML comment илрүүлнэ
  • DOMDocument-ээр parse хийж контент алга болсон эсэхийг шалгана
  • Эвдэрсэн HTML илэрвэл хадгалахгүй, алдааны мэдэгдэл харуулна
Анхааруулга: Source горимд HTML код засварлахдаа бүх tag-ийг зөв хааж, comment-ийг (<!-- ... -->) бүтэн бичихэд анхаарна уу. Эвдэрсэн HTML нь контентыг "залгидаг" тул шалгалт зайлшгүй шаардлагатай.

  • Код автоматаар 2-space indent-ээр форматлагдана
  • Cursor байрлал хоёр горим хооронд хадгалагдана
  • Ctrl+S дарж Visual горим руу буцна
