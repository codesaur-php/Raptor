What are Users?

Users is a module for managing system users. It handles user registration, editing, deletion, and all related operations.

  • Registration - Create new users and record their information
  • Edit - Modify user information
  • Delete - Deactivate users (soft delete)
  • Organization - Assign users to organizations
  • RBAC Role - Assign permission roles to users
  • Password - Password management

Edit

Click the edit button from the list.

  • All fields can be modified (except username)
  • Photo can be replaced or removed

Assign Organization

Users with system_user_organization_set permission can add or remove users from organizations.

Users with the system_coder role have access to all organizations.

Assign RBAC Role

Users with system_rbac permission can assign or remove roles from users. (See: RBAC Manual)

Users with the system_coder role have permission to perform all actions.

Password

  • Users can change their own password
  • Users with system_coder permission can change other users' passwords

Delete

Click the delete button from the list.

  • A SweetAlert confirmation dialog will appear
  • A text input field for providing a reason
  • Soft delete (deactivate) - data is preserved, the user is marked inactive
  • Deactivated users can be permanently deleted using the fire () button; permanently deleted data is stored in the Trash table for recovery
