What is Comments?

Comments is the Dashboard module for managing news article comments. You can view visitor comments, reply to them, and delete comments.

Comment features:

  • 1-level reply - Reply to root comments only (no nested replies)
  • Admin reply - Admin can reply to comments from the Dashboard
  • Guest / Admin distinction - Admin comments are marked with icon
  • News-linked - Comments depend on the news article's comment=1 setting

View News Comments

Click a news article from the list to view all its comments.

News Information

The top section shows the news article summary:

  • News title and published date
  • Photo (if available)
  • Publisher name, read count, publication status
  • Full article content
Comments Thread
  • Root comments and their replies in chronological order
  • Reply comments are indented with a blue left border
  • Admin comments show icon with a light background

Reply to Comment

Click the reply button next to a root comment.

  • A reply form opens showing who you are replying to
  • Write your reply and click Send
  • The reply appears under the original comment
  • Admin name is automatically set
Only root comments can be replied to. Replying to a reply is not allowed (1 level only).

Delete Comment

Click the delete (trash) button next to a comment.

  • A confirmation dialog appears (SweetAlert2)
  • Deleting a root comment also deletes all its replies
  • Permanently deleted from the database; a backup copy is stored in the Trash table for recovery

Admin Email Notification

An automatic email notification can be sent to an admin whenever a new comment is posted on a news article. This setting is located at the top of the list page (visible only to users with system_coder role).

Settings
SettingDescription
Enable / Disable Toggle switch ON = sets the email address, OFF = clears the email address

Notification emails include the commenter's name, email, comment content, and news article title.

This setting directly modifies the RAPTOR_COMMENT_EMAIL_TO value in the .env file. Email set = notifications on, empty = notifications off.
