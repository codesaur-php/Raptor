What is an Order?

Orders is the module for managing product orders received from the website.

The Orders module has the following features:

  • Order status workflow - New -> Processing -> Confirmed -> Shipped -> Completed
  • Email notifications - Automatic notifications sent to users
  • User information - Customer name and email address

Order List

Click Orders from the Dashboard menu to open the list page.

Filters

You can filter orders by the following criteria:

  • Language - By language code such as mn, en
  • Status - By order status (new, processing, confirmed, etc.)

View Order

Click the view (eye) button from the list.

  • Detailed order information - product, quantity, user information
  • Change status - select a new status from the dropdown
  • Send email to the customer

Notifications (Discord + Email)

Discord Notification

When an order status is changed, an automatic Discord notification is sent via webhook (if configured).

ActionNotification details
Status changeOrder ID, customer name, old status, new status, admin name
Admin Email Notification

An automatic email notification can be sent to an admin whenever a new order is received. This setting is located at the top of the list page (visible only to users with system_coder role).

SettingDescription
Enable / Disable Toggle switch ON = sets the email address, OFF = clears the email address
This setting directly modifies the RAPTOR_ORDER_EMAIL_TO value in the .env file. Email set = notifications on, empty = notifications off.
Customer Email Notification

Automatic emails are sent to the customer for the following events:

ActionRecipientTemplateContent
New order (from website) Customer order-confirmation Order ID, customer name, product, quantity, JWT confirmation link
Status change (from Dashboard) Customer order-status-update Order ID, old/new status, customer name, product

Delete Order

Click the delete (trash) button from the list.

  • A confirmation dialog appears (SweetAlert2)
  • Permanently deleted from the database; a backup copy is stored in the Trash table for recovery
  • system_product_delete permission is required
