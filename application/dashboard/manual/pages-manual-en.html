What is Pages?

Pages is the content management module of the Raptor Dashboard. It handles creating, editing, deleting, and publishing website pages.

Key features:

  • Tree structure - Pages can have parent-child hierarchy
  • Multi-language - Separate pages per language
  • SEO-friendly slug - Auto-generated URL slug from title
  • Navigation - Published pages with type menu or *-menu automatically appear in website menus
  • Special pages - Pages with type='special' are excluded from navigation menus but can appear in specific locations designated by the developer

Pages List

Click Pages in the Dashboard sidebar to open the list page.

Filters

Filter pages by:

  • Language - mn, en, etc.
  • Type - menu, mega-menu, special
  • Category - general, info, service, legal, reference
  • Status - Published or draft

Publishing Options

Users with system_content_publish permission see these options:

  • Publish - Make visible on the website
  • Feature - Show in the homepage featured section
  • Comments - Allow visitor comments
Auto-generated Fields
  • Slug - Auto-generated from title (Cyrillic to Latin transliteration)
  • Description - Auto-extracted from content (200 characters) if left empty
  • Position - Auto-calculated based on parent hierarchy

Edit Page

Click the edit (pencil) button from the list.

  • All fields can be modified (except slug)
  • Header image can be replaced or removed
  • Attachment descriptions can be edited, new files added, or existing files deleted
  • Editing a published page requires system_content_publish permission
Parent Page Selection
The parent page must share the same language. You cannot select a descendant page as a parent (prevents circular references).

View Page

Click the view (eye) button from the list.

  • All fields displayed in read-only mode
  • Content shown in moedit read-only mode
  • Header image and attachments visible
  • Warning shown if page has child pages
  • Warning shown if page is a link page

Delete Page

Click the delete (trash) button from the list.

  • A confirmation dialog appears (SweetAlert2)
  • Soft delete - data is preserved, only deactivated
  • Attachments and child pages remain intact
  • Requires system_content_delete permission

Link Pages

When the Link field is filled, the page acts as a redirect link instead of displaying content. Clicking it in the navigation menu redirects to that URL.

Supported link formats:

  • https://example.com - External URL
  • /about - Internal path
  • mailto:info@example.com - Email
  • tel:+97699001234 - Phone number

Reset Sample Data

When the system is first installed, sample data is pre-loaded. If sample data is detected, a yellow alert with a Reset & Production button appears on the list page.

Warning: Resetting will delete only sample data and their file attachment records. User-created content will remain intact. This action cannot be undone.
How sample data is detected: Pages with category='_raptor_sample_' are considered sample data. If you want to keep some of them as real content, simply change the category (e.g. to general) and save - they will not be deleted during reset.
